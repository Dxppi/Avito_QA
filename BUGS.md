## Баг 0
**Title:** Некорректный формат ответа API при создании объявления  
**Description:** Эндпоинт POST /api/1/item возвращает ответ в нарушение REST стандартов - строку статуса вместо JSON объекта с данными созданного объявления. Это вынуждает клиент парсить текст сообщения для получения ID.  
**Steps to Reproduce:**
1) Отправить POST запрос на /api/1/item с корректными данными
2) Получить ответ от сервера
3) Проанализировать формат ответа

**Expected Result:** API возвращает полный JSON объект созданного объявления со всеми полями (id, sellerId, name, price, createdAt, statistics)  
**Actual Result:** API возвращает JSON с полем status, содержащим текстовое сообщение с ID  
**Severity:** Major (Нарушение контракта API, усложнение клиентской логики)  
**Priority:** High

## Баг 1
**Title:** Дублирование функциональности эндпоинтов статистики  
**Description:** В API присутствуют два идентичных эндпоинта для получения статистики объявления (/api/1/statistic/{id} и /api/2/statistic/{id}), которые возвращают одинаковые данные. Это создает путаницу и нарушает принцип единственной ответственности.  
**Steps to Reproduce:**
1) Создать объявление и получить его ID
2) Отправить GET запрос на /api/1/statistic/{id}
3) Отправить GET запрос на /api/2/statistic/{id}
4) Сравнить ответы от обоих эндпоинтов

**Expected Result:** В API должен быть один канонический эндпоинт для получения статистики, либо разные версии должны иметь значимые отличия в функциональности  
**Actual Result:** Оба эндпоинта возвращают идентичные данные, создавая избыточность  
**Severity:** Minor (Технический долг, путаница в документации)  
**Priority:** Low

## Баг 2
**Title:** Несоответствие HTTP статус-кодов для несуществующих ресурсов  
**Description:** API возвращает различные статус-коды для операций с несуществующими объявлениями, нарушая стандарты HTTP. GET запросы возвращают 400 вместо 404, а DELETE всегда возвращает 200.  
**Steps to Reproduce:**
1) Отправить GET запрос на /api/1/item/nonexistent-id
2) Отправить GET запрос на /api/1/statistic/nonexistent-id  
3) Отправить DELETE запрос на /api/2/item/nonexistent-id
4) Проанализировать статус-коды ответов

**Expected Result:** Все операции с несуществующими ресурсами возвращают 404 Not Found  
**Actual Result:** GET возвращает 400 Bad Request, DELETE возвращает 200 OK даже для несуществующих items  
**Severity:** Major (Нарушение HTTP стандартов, путаница в обработке ошибок)  
**Priority:** High

## Баг 3
**Title:** Неконсистентность имен полей в запросах и ответах  
**Description:** Поле идентификатора продавца имеет разное написание в теле запроса (sellerID) и в ответах API (sellerId), что приводит к ошибкам при десериализации.  
**Steps to Reproduce:**
1) Отправить POST запрос с полем "sellerID"
2) Получить ответ с полем "sellerId" 
3) Попытаться десериализовать ответ используя ту же модель данных

**Expected Result:** Имена полей идентичны в запросах и ответах API  
**Actual Result:** sellerID (запрос) vs sellerId (ответ) - разный case  
**Severity:** Major (Ошибки десериализации, нарушение контракта)  
**Priority:** High

## Баг 4
**Title:** Отсутствие валидации бизнес-логики при создании объявлений  
**Description:** API принимает некорректные данные без валидации: пустые названия, отрицательные цены, отрицательные значения статистики, неверные типы данных.  
**Steps to Reproduce:**
1) Отправить POST запрос с пустым именем: "name": ""
2) Отправить POST запрос с отрицательной ценой: "price": -100
3) Отправить POST запрос с отрицательной статистикой: "likes": -10
4) Проверить что объявления создаются

**Expected Result:** API должен возвращать 400 Bad Request для некорректных бизнес-данных  
**Actual Result:** Объявления создаются с некорректными данными  
**Severity:** Major (Коррупция данных, нарушение бизнес-правил)  
**Priority:** Medium

## Баг 5
**Title:** Нарушение целостности данных при удалении объявлений  
**Description:** После удаления объявления через DELETE /api/2/item/{id} статистика объявления остается доступной.  
**Steps to Reproduce:**
1) Создать объявление и запомнить ID
2) Удалить объявление через DELETE /api/2/item/{id}
3) Проверить что GET /api/1/item/{id} возвращает 404
4) Проверить что GET /api/1/statistic/{id} возвращает данные

**Expected Result:** После удаления объявления статистика также должна быть недоступна (404)  
**Actual Result:** Статистика остается доступной после удаления объявления  
**Severity:** Major (Нарушение ссылочной целостности, утечка данных)  
**Priority:** Medium
